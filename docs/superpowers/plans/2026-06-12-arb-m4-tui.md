# M4 — TUI Dashboard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the ratatui dashboard (spec §17) on top of the M3 paper-trading session: five live panels (opportunity feed, positions & P&L, fills/orders, system health, scrolling log), keyboard controls (pause, paper/live modal, kill switch, quit), fed by a `watch<Arc<AppState>>` publisher at ~10 Hz that can never block producers — plus the M3-carry-over mid-mark calibration for the drawdown halt.

**Architecture:** A new presentation-only crate `pm-tui` (pm-core + ratatui/crossterm + tokio-sync) renders display-ready plain data (`AppState`) and emits `TuiCommand`s; it holds no business logic. An app-side publisher task assembles `AppState` every ~100 ms from four non-blocking sources: `AppStats` atomics, ingestion `StatsCell` gauges, a second READ-ONLY sqlite connection (WAL allows concurrent readers; the single writer task stays sole writer), and a small `watch<CoordStatus>` the coordinator publishes. The coordinator gains a control channel (pause) reusing the supervisor's Option-receiver select pattern, and computes BOTH bid-marks (reporting, durable rows — unchanged) and mid-marks (feeds the drawdown halt, fixing the live-run artifact where every open basket dipped equity by spread+fees). The `arb` binary defaults to the dashboard on a tty with `--headless` preserving M3 behavior; in TUI mode tracing tees into a ring buffer instead of stdout, and a panic hook restores the terminal.

**Tech Stack:** ratatui 0.29 + crossterm 0.28 (TestBackend for render tests; blocking input reader on a std thread), tokio watch/mpsc, rusqlite read-only connection. No new business-logic dependencies.

**Spec:** `docs/superpowers/specs/2026-06-12-polymarket-arb-bot-v2-design.md` §17 (panels, keys, watch input, never-block rule), §3 (tui crate), §20 (p50/p99 on the TUI), §22 M4 exit: "full dashboard on live paper session".

## Design decisions locked in this plan

1. **TUI reads the STORE, not the pipeline** (final-review M4 note). Opportunity feed, fills, order states, positions, and P&L history come from `ReadStore` LIMIT-queries at 10 Hz — the durable truth, zero new producer-side plumbing, structurally incapable of blocking producers (spec §17). Live counters/latencies come from the existing `Arc<AppStats>`/`StatsCell` atomics. Coordinator-only state (paused/halted/killed/busy + last P&L incl. the new mid equity) comes via a `watch<CoordStatus>`.
2. **Two equities, labeled** (M3 final-review recommendation): bid-marked equity stays the durable/reported number (pnl_snapshots rows unchanged); a NEW mid-marked equity feeds `risk.update_equity` so the 2% drawdown halt stops hair-triggering on spread+fees of open baskets. The dashboard shows both ("equity (bid)" / "risk equity (mid)").
3. **"Pause scanning" = dispatch pause.** The key drives `risk.set_paused` through a coordinator control channel; detection keeps running (books stay warm, opportunities still logged with `dispatched=false`). Footer labels it `[p]ause dispatch` — honest naming.
4. **Paper/live toggle is an M5 stub behind the full modal.** The typed-confirmation modal (spec §17) is fully implemented; confirming "live" displays "live venue unavailable until M5" and stays in paper. The modal machinery is the M4 deliverable; the venue switch is M5's.
5. **Kill from the TUI sets the existing `Arc<AtomicBool>`** (same flag the sentinel file/SIGUSR1 watcher sets) after a y/N confirm modal — one kill path, no new states. Quit (`q`) ends the session gracefully (same as duration-elapsed).
6. **Input on a std thread** (blocking `crossterm::event::read()` → mpsc) instead of the async EventStream feature — fewer deps, and `handle_key` stays a pure, fully unit-testable function of `(KeyEvent, &mut UiState) -> Option<TuiCommand>`.
7. **Tracing in TUI mode tees to a ring buffer, NOT stdout** (stdout writes corrupt the alternate screen). Headless mode keeps the M3 stdout fmt layer exactly as-is. The ring layer stores pre-formatted lines; the log panel renders the tail.
8. **Solver queue depth** (spec §17 health item) is derived as `lp_jobs − lp_solved − lp_dropped_full` from a new `lp_solved` counter — no channel-capacity introspection, no extra Sender clones that would break the shutdown cascade.
9. **Market display names**: the registry starts retaining the Gamma question text (it already flows into `RegistryBuilder::add_market`; M3 discarded it after partition screening). Publisher resolves token → market → question (fallback: truncated condition id). No store schema change.
10. **Terminal safety**: raw mode + alternate screen wrapped with (a) a panic hook that restores the terminal BEFORE the default hook prints, and (b) teardown before the final session report so exit-code logs land on the normal screen.
11. **Frames/s** is computed by the publisher from successive `StatsCell` frame counters over the refresh interval — no producer changes.

```rust
//! Read-only dashboard queries (spec §17). A SECOND sqlite connection opened
//! with SQLITE_OPEN_READ_ONLY: WAL allows it to read concurrently with the
//! single writer task, and read-only mode makes blocking the writer
//! structurally impossible.

use std::path::Path;

use rusqlite::{Connection, OpenFlags};

use crate::{PnlRow, StoreError};

/// One opportunity row, display-ready.
#[derive(Debug, Clone)]
pub struct OppView {
    pub ts_ms: i64,
    pub class: String,
    pub edge_bps: i64,
    pub units_micro: i64,
    pub net_micro: i64,
    pub legs_json: String,
    pub dispatched: bool,
}

#[derive(Debug, Clone)]
pub struct FillView {
    pub ts_ms: i64,
    pub token: i64,
    pub action: String,
    pub px_ticks: i64,
    pub tick_levels: i64,
    pub qty_micro: i64,
    pub cash_micro: i64,
}

#[derive(Debug, Clone)]
pub struct OrderEventView {
    pub ts_ms: i64,
    pub order_id: String,
    pub state: String,
    pub detail: String,
}

#[derive(Debug, Clone)]
pub struct HaltView {
    pub ts_ms: i64,
    pub reason: String,
}

pub struct ReadStore {
    conn: Connection,
}

impl ReadStore {
    /// Open read-only. Errors if the database file does not exist (the writer
    /// creates it; the dashboard must never create an empty shadow db).
    pub fn open(path: &Path) -> Result<Self, StoreError> {
        let conn = Connection::open_with_flags(
            path,
            OpenFlags::SQLITE_OPEN_READ_ONLY | OpenFlags::SQLITE_OPEN_NO_MUTEX,
        )?;
        // Bounded wait if the writer holds the lock during a checkpoint.
        conn.busy_timeout(std::time::Duration::from_millis(100))?;
        Ok(ReadStore { conn })
    }

    pub fn recent_opportunities(&self, n: usize) -> Result<Vec<OppView>, StoreError> {
        let mut stmt = self.conn.prepare(
            "SELECT ts_ms, class, edge_bps, units_micro, net_micro, legs_json, dispatched
             FROM opportunities ORDER BY id DESC LIMIT ?1",
        )?;
        let rows = stmt
            .query_map([n as i64], |r| {
                Ok(OppView {
                    ts_ms: r.get(0)?,
                    class: r.get(1)?,
                    edge_bps: r.get(2)?,
                    units_micro: r.get(3)?,
                    net_micro: r.get(4)?,
                    legs_json: r.get(5)?,
                    dispatched: r.get(6)?,
                })
            })?
            .collect::<Result<Vec<_>, _>>()?;
        Ok(rows)
    }

    pub fn recent_fills(&self, n: usize) -> Result<Vec<FillView>, StoreError> {
        let mut stmt = self.conn.prepare(
            "SELECT ts_ms, token, action, px_ticks, tick_levels, qty_micro, cash_micro
             FROM fills ORDER BY id DESC LIMIT ?1",
        )?;
        let rows = stmt
            .query_map([n as i64], |r| {
                Ok(FillView {
                    ts_ms: r.get(0)?,
                    token: r.get(1)?,
                    action: r.get(2)?,
                    px_ticks: r.get(3)?,
                    tick_levels: r.get(4)?,
                    qty_micro: r.get(5)?,
                    cash_micro: r.get(6)?,
                })
            })?
            .collect::<Result<Vec<_>, _>>()?;
        Ok(rows)
    }

    pub fn recent_order_events(&self, n: usize) -> Result<Vec<OrderEventView>, StoreError> {
        let mut stmt = self.conn.prepare(
            "SELECT ts_ms, order_id, state, detail
             FROM order_events ORDER BY id DESC LIMIT ?1",
        )?;
        let rows = stmt
            .query_map([n as i64], |r| {
                Ok(OrderEventView {
                    ts_ms: r.get(0)?,
                    order_id: r.get(1)?,
                    state: r.get(2)?,
                    detail: r.get(3)?,
                })
            })?
            .collect::<Result<Vec<_>, _>>()?;
        Ok(rows)
    }

    /// Open positions from FIFO lots: (token, qty µshares, cost basis µUSDC).
    pub fn open_positions(&self) -> Result<Vec<(i64, i64, i64)>, StoreError> {
        let mut stmt = self.conn.prepare(
            "SELECT token, SUM(remaining_micro), SUM(cost_remaining_micro)
             FROM lots GROUP BY token HAVING SUM(remaining_micro) > 0 ORDER BY token",
        )?;
        let rows = stmt
            .query_map([], |r| Ok((r.get(0)?, r.get(1)?, r.get(2)?)))?
            .collect::<Result<Vec<_>, _>>()?;
        Ok(rows)
    }

    pub fn latest_pnl(&self) -> Result<Option<PnlRow>, StoreError> {
        let mut stmt = self.conn.prepare(
            "SELECT ts_ms, cash_micro, realized_micro, unrealized_micro, equity_micro
             FROM pnl_snapshots ORDER BY id DESC LIMIT 1",
        )?;
        let mut rows = stmt.query_map([], |r| {
            Ok(PnlRow {
                ts_ms: r.get(0)?,
                cash_micro: r.get(1)?,
                realized_micro: r.get(2)?,
                unrealized_micro: r.get(3)?,
                equity_micro: r.get(4)?,
            })
        })?;
        Ok(rows.next().transpose()?)
    }

    pub fn recent_halts(&self, n: usize) -> Result<Vec<HaltView>, StoreError> {
        let mut stmt = self.conn.prepare(
            "SELECT ts_ms, reason FROM halts ORDER BY id DESC LIMIT ?1",
        )?;
        let rows = stmt
            .query_map([n as i64], |r| Ok(HaltView { ts_ms: r.get(0)?, reason: r.get(1)? }))?
            .collect::<Result<Vec<_>, _>>()?;
        Ok(rows)
    }

    /// Test hook: raw connection access for the read-only enforcement test.
    #[cfg(test)]
    pub(crate) fn conn_for_test(&self) -> &Connection {
        &self.conn
    }
```

```rust
//! Dashboard rendering (spec §17): dark theme; green/red reserved for edge
//! and P&L. Pure function of (&AppState, &UiState) — no I/O, no business logic.

use ratatui::layout::{Constraint, Direction, Layout, Rect};

/// Money color: green ≥ 0, red < 0 (the ONLY places green/red are used are
/// edge and P&L values, per spec §17).
fn money_style(v: f64) -> Style {
    if v < 0.0 {
        Style::default().fg(Color::Red)
    } else {
        Style::default().fg(Color::Green)
    }
}

fn fmt_usd(v: f64) -> String {
    format!("{v:.2}")
}

fn fmt_uptime(s: u64) -> String {
    if s >= 3600 {
        format!("{}:{:02}:{:02}", s / 3600, (s % 3600) / 60, s % 60)
    } else {
        format!("{:02}:{:02}", s / 60, s % 60)
    }
}

/// Top-level draw. Layout:
/// header(3) / upper panels(40%) / lower panels(40%) / log(rest) / footer(1).
pub fn draw(f: &mut Frame, s: &AppState, ui: &UiState) {
    let outer = Layout::default()
        .direction(Direction::Vertical)
        .constraints([
            Constraint::Length(3),
            Constraint::Percentage(32),
            Constraint::Percentage(28),
            Constraint::Min(4),
            Constraint::Length(1),
        ])
        .split(f.area());

    draw_header(f, s, outer[0]);

    let upper = Layout::default()
        .direction(Direction::Horizontal)
        .constraints([Constraint::Percentage(62), Constraint::Percentage(38)])
        .split(outer[1]);
    draw_opportunities(f, s, upper[0]);
    draw_positions(f, s, upper[1]);

    let lower = Layout::default()
        .direction(Direction::Horizontal)
        .constraints([Constraint::Percentage(62), Constraint::Percentage(38)])
        .split(outer[2]);
    draw_fills_orders(f, s, lower[0]);
    draw_health(f, s, lower[1]);

    draw_log(f, s, ui, outer[3]);
    draw_footer(f, s, outer[4]);

    if ui.modal != Modal::None {
        draw_modal(f, ui);
    }
}

fn draw_header(f: &mut Frame, s: &AppState, area: Rect) {
    let mode = if s.mode_paper { "PAPER" } else { "LIVE" };
    let mut badges: Vec<Span> = vec![

    if s.paused {
        badges.push(Span::styled(" PAUSED ", Style::default().add_modifier(Modifier::REVERSED)));
    }

fn centered_rect(pct_x: u16, pct_y: u16, r: Rect) -> Rect {

            Constraint::Percentage((100 - pct_y) / 2),

        .direction(Direction::Horizontal)
        .constraints([
            Constraint::Percentage((100 - pct_x) / 2),
            Constraint::Percentage(pct_x),
            Constraint::Percentage((100 - pct_x) / 2),
        ])
        .split(v[1])[1]
}
```

```rust
/// Pure key → command mapping. `paused` is the CURRENT pause state (the
/// toggle emits its inverse). Modal-first: when a modal is open, keys edit
/// the modal and never leak global bindings.
pub fn handle_key(ev: KeyEvent, ui: &mut UiState, paused: bool) -> Option<TuiCommand> {
    // Ctrl-C always quits, modal or not.
    if ev.modifiers.contains(KeyModifiers::CONTROL) && ev.code == KeyCode::Char('c') {
        return Some(TuiCommand::Quit);
    }
    match &mut ui.modal {
        Modal::ConfirmKill => {
            match ev.code {
                KeyCode::Char('y') | KeyCode::Char('Y') => {
                    ui.modal = Modal::None;
                    return Some(TuiCommand::Kill);
                }
                KeyCode::Char('n') | KeyCode::Char('N') | KeyCode::Esc => ui.modal = Modal::None,
                _ => {}
            }
            None
        }
        Modal::ConfirmLive(typed) => {
            match ev.code {
                KeyCode::Enter if typed == "live" => {
                    ui.modal = Modal::None;
                    return Some(TuiCommand::GoLive);
                }
                KeyCode::Enter => {} // wrong text: stay for correction
                KeyCode::Backspace => {
                    typed.pop();
                }
                KeyCode::Esc => ui.modal = Modal::None,
                KeyCode::Char(c) => typed.push(c),
                _ => {}
            }
            None
        }

            KeyCode::Char('p') => Some(TuiCommand::SetPaused(!paused)),
```

```rust
//! The TUI event loop: redraw on AppState changes (~10 Hz) and on key input;
//! emit TuiCommands. Input is read on a dedicated std thread (blocking
//! crossterm::event::read) and forwarded over an mpsc — handle_key stays pure.

use std::sync::Arc;

use crossterm::event::{Event, KeyEvent, KeyEventKind};
use ratatui::backend::Backend;
use ratatui::Terminal;
use tokio::sync::{mpsc, watch};

use crate::render::draw;
use crate::state::{AppState, TuiCommand};
use crate::ui::UiState;

/// Spawn the blocking input reader. Exits when the receiver is dropped.
pub fn spawn_input_thread() -> mpsc::Receiver<KeyEvent> {
    let (tx, rx) = mpsc::channel(32);
    std::thread::spawn(move || {
        loop {
            match crossterm::event::read() {
                Ok(Event::Key(k)) if k.kind == KeyEventKind::Press => {
                    if tx.blocking_send(k).is_err() {
                        return; // TUI gone
                    }
                }
                Ok(_) => {}     // resize/mouse: next draw picks it up
                Err(_) => return,
            }
        }
    });
    rx
}

/// Drive the dashboard until Quit is emitted or the state channel closes.
/// The caller owns terminal setup/teardown (raw mode, alternate screen).
pub async fn run_tui<B: Backend>(
    terminal: &mut Terminal<B>,
    mut state_rx: watch::Receiver<Arc<AppState>>,
    mut key_rx: mpsc::Receiver<KeyEvent>,
    cmd_tx: mpsc::Sender<TuiCommand>,
) -> std::io::Result<()> {
    let mut ui = UiState::default();
    loop {
        let state = state_rx.borrow_and_update().clone();
        terminal.draw(|f| draw(f, &state, &ui))?;
        tokio::select! {
            changed = state_rx.changed() => {
                if changed.is_err() {
                    return Ok(()); // publisher gone: session ending
                }
            }
            maybe_key = key_rx.recv() => {
                let Some(key) = maybe_key else { return Ok(()) };
                let paused = state.paused;
                if let Some(cmd) = crate::ui::handle_key(key, &mut ui, paused) {
                    let quit = cmd == TuiCommand::Quit;
                    let _ = cmd_tx.send(cmd).await;
                    if quit {
                        return Ok(());
                    }
                }
            }
        }
    }
}
```

```rust
/// (level code, formatted line); level: 1=ERROR 2=WARN 3=INFO 4=DEBUG 5=TRACE.

    cap: usize,
}

    pub fn new(cap: usize) -> Arc<Self> {
        Arc::new(LogBuffer { lines: Mutex::new(VecDeque::with_capacity(cap)), cap: cap.max(1) })
    }

    fn push(&self, lvl: u8, line: String) {
        if let Ok(mut q) = self.lines.lock() {
```

```rust
/// 10 Hz publish loop. Ends when the coordinator's status channel closes

pub fn spawn_publisher(
    mut ctx: PublisherCtx,
    refresh: std::time::Duration,
) -> (watch::Receiver<Arc<AppState>>, tokio::task::JoinHandle<()>) {
    let (tx, rx) = watch::channel(Arc::new(AppState::default()));
    let handle = tokio::spawn(async move {
        let mut tick = tokio::time::interval(refresh);
        tick.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Skip);
        loop {
            tick.tick().await;
            if ctx.status_rx.has_changed().is_err() {
                break; // coordinator gone
            }
            let state = assemble(&mut ctx).await;
            if tx.send(Arc::new(state)).is_err() {
                break; // no TUI receivers left
            }
        }
    });
    (rx, handle)
}
```
